# 考研资源整合与学习规划系统（Kaoyan Guide）

一个面向考研学生的智能化学习与信息服务平台。  
项目采用 **Spring Boot + Vue 3** 前后端分离架构，围绕“**院校信息获取 + AI 智能咨询 + 个性化学习规划 + 日常学习支持**”构建完整业务闭环。

它不仅提供传统的院校库、专业库、招生政策、公告资讯等查询能力，还结合 **大语言模型（LLM）** 与 **RAG（检索增强生成）**，将“信息展示系统”升级为“可对话、可规划、可持续使用”的考研辅助平台。

---

## 项目亮点

### 1. AI 智能院校咨询
系统将大模型问答能力与本地院校/专业数据结合，支持围绕考研院校、专业方向、报考建议等内容进行智能咨询。  
相比普通搜索，本项目更强调：

- 基于本地知识与业务数据回答问题
- 支持多轮对话
- 支持流式输出，提升交互体验
- 可根据模块切换不同提示词、上下文与工具能力

### 2. AI 个性化学习规划
系统可结合用户近几天的学习记录与反馈，自动生成当天学习计划，包括：

- 学习建议
- 分学科任务拆解
- 任务状态管理
- 计划的持续迭代与调整

这使平台不仅能“回答问题”，还能“持续陪伴学习”。

### 3. 完整的考研信息资源体系
项目内置多类核心数据资源，支持学生围绕考研关心的信息进行集中浏览与查询，例如：

- 院校信息
- 专业门类与专业详情
- 招生政策
- 专业解读
- 系统公告
- 学校评价与收藏

### 4. 学生端 + 管理端双端设计
系统同时包含：

- **学生端**：面向考研用户的信息查询、AI 咨询、学习规划、模拟考试等功能
- **管理端**：面向管理员的数据维护、题库管理、公告管理、院校管理、数据统计等功能

### 5. 业务闭环完整，适合作为毕业设计项目展示
从“信息管理”到“智能问答”，再到“学习规划”和“学习支持”，本项目不只是 CRUD 系统，而是具备较强业务连贯性和一定创新点的综合型毕设项目。

## 核心业务逻辑

## 1. AI 智能院校咨询模块逻辑

这是项目最关键的智能模块之一，整体流程可以概括为：

### 第一步：前端发起咨询请求

学生在院校咨询页面输入问题后，前端调用统一聊天接口，并携带：

- 用户问题
- 模块类型 `consult_college`
- 会话标识 `sessionId`

### 第二步：后端统一聊天入口接收请求

后端通过统一的 `/chat` 接口处理请求，完成：

- 登录状态校验
- 会话 ID 生成或复用
- SSE 流式响应建立
- 将请求路由到对应业务模块

### 第三步：聊天网关按模块分发

系统通过聊天网关统一处理不同 AI 场景：

- `consult_college`：院校咨询模块
- `learning_plan`：学习规划模块

网关会继续完成：

- 构造 memoryId
- 判断是否启用会话记忆
- 读取历史消息
- 选择提示词资源
- 选择是否启用 RAG
- 选择是否启用工具调用

### 第四步：RAG + 大模型生成答案

在院校咨询场景下，系统将：

- 用户问题交给大模型
- 结合本地知识检索内容
- 结合预设提示词约束回答风格
- 在需要时调用工具能力
- 最终以流式方式返回给前端

### 第五步：流式输出与结果清洗

模型返回内容后，后端会对输出做统一清洗，再将分片内容逐步推送给前端，实现更自然的实时对话体验。

### 这一模块的价值

该模块不是简单调用聊天接口，而是加入了：

- 模块化路由
- 会话记忆
- 提示词资源化
- RAG 检索增强
- 流式输出
- 工具扩展能力

因此它是整个项目最有“智能系统”特征的部分。

## 2. 智能学习规划模块逻辑

这是另一个非常有代表性的核心模块，重点不在“问答”，而在“生成结构化学习计划”。

### 第一步：读取指定日期计划

系统支持根据日期查询用户学习计划。
 若当天已有已生成计划，则直接返回，避免重复生成。

### 第二步：拉取历史学习记录

当用户生成今日计划时，系统会自动读取最近几天（如前 3 天）的学习历史，包括：

- 当天任务内容
- 完成情况
- 用户反馈

### 第三步：拼接学习上下文

系统将历史记录整理为可供大模型理解的上下文文本，并与“昨日反馈/当前状态”一起拼接成完整提示词。

### 第四步：调用 AI 生成结构化结果

学习规划模块会明确约束模型：

- 只输出 JSON
- 输出一段学习建议 `advice`
- 输出任务列表 `tasks`
- 每个任务包含科目、内容、完成状态等字段

### 第五步：解析与清洗 AI 返回结果

系统会对模型返回内容进行：

- JSON 清洗
- 格式校验
- 任务标准化处理

如果 AI 返回格式异常，则进行异常拦截，避免错误数据直接入库。

### 第六步：计划入库

系统将生成结果拆成两层结构保存：

- 主计划：保存日期、建议、反馈、计划状态等
- 子任务：保存每条任务的科目、内容、来源、完成状态、排序等

### 第七步：前端展示与后续迭代

前端页面会展示：

- 今日建议
- 今日任务清单
- 已完成 / 未完成状态
- 用户后续反馈

这样第二天生成新计划时，系统又能继续参考这些历史记录，形成连续学习闭环。

### 这一模块的价值

它体现了项目从“信息系统”走向“智能辅助系统”的另一个方向：
 不仅回答问题，还能基于历史行为做任务级别的学习安排。

## 快速开始

## 1. 环境准备

建议准备以下环境：

- JDK 17
- Node.js 16+
- MySQL 5.7 / 8.0
- Redis 6+
- Maven 3.8+

## 2. 数据库初始化

执行根目录下的数据库脚本：

```
kaoyan_guide.sql
```

创建数据库并导入表结构及初始化数据。

## 3. 启动后端

进入后端工程：

```
cd kaoyan_guide_springboot
```

配置 `application.yml` 中的：

- MySQL 连接信息
- Redis 连接信息
- 大模型 API 配置
- 文件上传目录配置

启动 Spring Boot 主程序即可。

## 4. 启动前端

进入前端工程：

```
cd kaoyan_guide_vue
npm install
npm run dev
```

启动后按终端提示访问前端地址。
